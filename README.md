Описание реализации проекта
Задачи

    Добавить новый источник данных (API курьерской службы)

    Построить новую витрину для расчётов с курьерами

    Связать новый источник с уже существующими данными о заказах

Что сделано

В уже работающее хранилище данных (DWH) добавлен новый источник — API курьерской службы.
Из API загружены:

    список курьеров

    информация о доставках

    рейтинги

    чаевые

Эти данные связаны с уже существующими данными о заказах.

На основе объединённых данных построена новая витрина:

cdm.dm_courier_ledger — расчёт сумм выплат курьерам за предыдущий месяц.
В витрине учтены:

    количество и сумма заказов

    средний рейтинг курьера

    комиссия за обработку заказов (25%)

    выплата курьеру в зависимости от рейтинга: от 5% до 10% суммы заказа с минимальными порогами

    чаевые с учётом комиссии 5%

Инструменты
Компонент	Технология
База данных	PostgreSQL (или аналогичная реляционная БД)
Оркестрация	Apache Airflow (набор DAG)
API	вызовы через curl внутри Python-операторов Airflow, постраничная загрузка с пагинацией
Слои данных	STG (сырые JSON) → DDS (нормализованные таблицы, «снежинка») → CDM (готовая витрина)
Как организована работа

    Старые DAG по загрузке заказов продолжают работать без изменений

    Новый DAG:

        читает API курьеров за последние 7 дней

        складывает данные в STG

        переносит их в DDS

    Существующий DAG по заказам дополнен:

        теперь он подтягивает из DDS данные о курьерах и доставках

    Финальный этап (витрина):

        пересчитывает выплаты за прошлый месяц

        использует дату заказа, а не доставки

        это корректно обрабатывает ночные доставки на следующий день

Результат

Бизнес может на 10-е число каждого месяца брать из витрины готовые суммы для выплат курьерам. Все данные актуальны, логика выплат полностью автоматизирована.
