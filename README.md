Описание реализации проекта

Что сделано:
В уже работающее хранилище данных (DWH) добавлен новый источник — API курьерской службы. Данные из этого API (список курьеров, информация о доставках, рейтингах и чаевых) загружены и связаны с уже существующими данными о заказах.

На основе объединённых данных построена новая витрина cdm.dm_courier_ledger, которая рассчитывает суммы выплат курьерам за предыдущий месяц. В витрине учтены:

    количество и сумма заказов;

    средний рейтинг курьера;

    комиссия за обработку заказов (25%);

    выплата курьеру в зависимости от рейтинга (от 5% до 10% суммы заказа, но с минимальными порогами);

    чаевые с учётом комиссии 5%.

Инструменты:

    База данных: PostgreSQL (или аналогичная реляционная БД)

    Оркестрация: Apache Airflow (набор DAG)

    API: вызовы через curl внутри Python-операторов Airflow, постраничная загрузка с пагинацией

    Слои данных: STG (сырые JSON из API), DDS (нормализованные таблицы — «снежинка»), CDM (готовая витрина)

Как организована работа:

    Старые DAG, загружающие заказы, продолжают работать без изменений.

    Новый DAG читает API курьеров за последние 7 дней, складывает в STG, затем переносит в DDS.

    Существующий DAG по заказам дополнен: теперь он подтягивает из DDS данные о курьерах и доставках.

    Финальный этап (витрина) пересчитывает выплаты за прошлый месяц относительно даты заказа (не доставки), что корректно обрабатывает ночные доставки на следующий день.

Результат: Бизнес может на 10-е число каждого месяца брать из витрины готовые суммы для выплат курьерам.
